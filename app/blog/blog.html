What is MVC?

Simply put, MVC is a method for organizing code to make it scalable. Specifically, here are the three parts of MVC:

Model – The model is essentially the data used to populate the application. For example, logic that is related to a database would be the model component of an MVC app.

View – The view is what the user sees. For example, HTML files that display in a web browser would be the view component of an MVC app.

Controller – The controller is the bridge between the model and the view. It takes input/requests from the user interacting with the view and finds what is needed from the model to serve back to the user.

It may help to see a simple example of what MVC looks like in real code. For this example, here is what the three parts of our MVC are going to look like:

Model– Our model is going be a “database” that contains a list of customers. To keep this example as simple as possible, our “database” is simply going to be an array of objects.

View – Our view is going to consist of just a form with an input box and a button that will allow users to search the “database” (our model) for a customer.

Controller – Our controller is going to handle the search request from the view, search the “database” for the customer requested, and then serve back to the view what the “database”, or model, returns.
